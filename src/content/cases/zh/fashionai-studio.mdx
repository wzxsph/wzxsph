## 真正的产品，是完整证据链

生成一张图片只是工作流中的一个事件。合作方还需要知道：输入了哪些商品事实、使用了什么源素材、队列实际产出了什么、自动规则发现了什么、最终由谁选中了结果，以及交付包里究竟包含哪些文件。

FashionAI Studio 把这些判断逐项露出来，而不是把它们压缩成一句“AI 已完成”。

> Brief → Design → Generate → Results → Review → Deliver。六个阶段都能自由回看，缺少前置条件时就在当前页面说明原因。

## 六个阶段，各自承担一种责任

1. **Brief / 商品简报。** 在生成前记录模式、目标市场、商品事实和禁止声明。
2. **Design / 素材设计。** 上传正反视图、复用模板，并在确定性画布上放置透明标记。
3. **Generate / 生成方案。** 在极速三图链路和自定义交付素材包之间做选择。
4. **Results / 生成结果。** 查看真实队列状态、候选结果、单项下载与独立重试。
5. **Review / 人工审核。** 阅读可执行的 `pass`、`warn`、`fail` 证据，再明确做出人工选择；风险结果绝不会默认勾选。
6. **Deliver / 交付打包。** 只导出人工选中项，同时预览目录并生成 `manifest.json`。

当前阶段写入 URL，审核选择则按项目保存在 `sessionStorage`。桌面端使用主画布与粘性 Inspector；390px 手机端保留同一条完整链路，并改为折叠 Inspector 与粘性底部操作栏。

## 一套贯穿输入到交付的原创样例

公开 Mock 现在只使用一套连贯的无品牌编辑样例：石墨色服装输入、透明几何标记、白底商品候选、虚构模特候选和竖版社媒候选。

每个公开文件都附带尺寸和 SHA-256 校验和；仓库同时公开生成说明、提示词与通俗的 AI 声明。这些文件是产品演示证据，不是客户案例，也不代表真实商业成绩。

## 三种信号，界面绝不混为一谈

| 信号 | 它能证明什么 | 它不能证明什么 |
| --- | --- | --- |
| 任务成功 | 配置的生成任务返回了输出。 | 输出已经准确或可以发布。 |
| 质量结论 | 可执行规则留下了 `pass`、`warn` 或 `fail` 观察。 | 结果已经通过人工审批。 |
| 人工选择 | 审核者明确把该候选纳入本次交付。 | 授权与商品事实不再需要最终核验。 |

这一区分是新版最重要的交互决定。重试、风险提示与最终交付因此能同时被产品和工程审阅者看懂。

## 把边界和结果一起交付

Mock 数据仍只保存在浏览器本地，不会调用付费模型。真实 API、认证、数据库、队列、质量与导出契约继续兼容旧实现。任务只有记录了真实成本才展示数字，否则明确显示“未配置”；输入素材权利和最终商业审核仍由操作方负责。

最终得到的是一个更可信的合作入口：任何人都可以打开公开产品、走完完整链路、核验演示素材，并基于真实证据讨论下一步生产化，而不需要猜测哪些部分只是概念。
